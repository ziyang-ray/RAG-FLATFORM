# MP Agent 助手门户系统 — 功能清单

---

## 一、认证与用户管理

1. **用户登录**：基于用户名+密码的认证机制，使用 HMAC-SHA256 签名的自定义 JWT Token（非标准 JWT），Token 有效期默认 8 小时。
2. **密码安全**：密码使用 PBKDF2-SHA256 算法加盐哈希存储（12 万次迭代）。
3. **自动登录/Token 持久化**：前端将 Token 存储在 `localStorage`，页面刷新后自动恢复登录状态；若 Token 过期则自动清除并退回登录页。
4. **退出登录**：右上角账户下拉菜单提供退出按钮，清除本地 Token 并返回登录页。
5. **用户信息展示**：登录后右上角显示当前用户登录 ID。
6. **预置演示账号**：
   - 上帝账号：`MP / 12345678`（总部管理员，拥有所有部门权限）
   - 部门管理员：`MP-Q-ADMIN / 12345678`（质量部门管理员）
   - 普通员工：`MP-Q-STAFF / 12345678`（质量部门普通成员）
   - 另有 AT、PLM、AI、MD 各部门的 admin 和 staff 账号。

---

## 二、部门/组织管理

1. **多部门架构**：系统预设 6 个部门（MP、MP-Q、MP-AT、MP-PLM、MP-AI、MP-MD）。
2. **部门层级**：所有子部门的父部门为 `dept_mp`（MP 总部）。
3. **用户-部门多对多关系**：一个用户可以属于多个部门，每个部门有独立的角色，并有一个默认部门。
4. **部门切换**：上帝账号可在左侧面板切换当前操作部门；普通用户固定为默认部门。
5. **部门标签系统**：前端为每个部门显示不同颜色的标签（MP=蓝色、Q=绿色、PLM=紫色、AI=橙色、MC=青色、AT=红色）。

---

## 三、角色与权限体系（RBAC）

1. **三级角色**：
   - **上帝账号（MP）**：全局管理员，可管理所有部门的所有资源，可删除任意知识库。
   - **部门管理员（dept_admin）**：可管理本部门资源策略，审批分享申请，可删除非公开知识库。
   - **普通员工（member）**：只能管理自己的私密知识库，可查看本部门和公共资源。
2. **资源可见性策略**：
   - `public`：所有人均可见。
   - `dept`：仅所属部门成员可见。
   - `private`：仅创建者本人可见（部门管理员也不可见）。
3. **细粒度权限控制**：支持白名单部门、黑名单部门、白名单用户、白名单角色。
4. **资源分类**：知识库可标记为 A/B/C/D 等分类。

---

## 四、智能体（Agent）管理

1. **智能体列表**：从 RAGFlow 后端同步所有可用智能体，根据用户部门和权限过滤。
2. **智能体策略初始化**：管理员可一键根据名称推断归属部门和可见性。
3. **管理员策略编辑**：管理员可修改任意智能体的归属部门、可见性、分类等策略。
4. **"不使用智能体"模式**：可直接调用 GPT-5.4 大模型进行对话（直连模式）。

---

## 五、知识库管理

### 5.1 上传知识库
1. **创建并上传**：输入名称、描述，选择多个文件上传，创建属于当前部门的知识库。
2. **私密知识库选项**：上传时可勾选"私密知识库"，仅创建者可见。
3. **自动解析**：文件上传后自动触发 RAGFlow 异步文档解析。
4. **命名规则**：知识库名称自动添加部门前缀（如 `MP-Q-我的知识库`）。

### 5.2 分享知识库
5. **部门内共享**：普通员工可将自己的私密知识库共享为部门可见级别。
6. **跨部门分享申请**：管理员可发起分享申请（公开或指定人员）。
7. **申请理由**：提交分享申请时可附带理由说明。
8. **分享申请审批流程**：管理员可批准或驳回申请，可附加审批意见，批准后自动更新可见性。
9. **待审批徽章**：侧边栏"分享申请"按钮旁显示待审批数量。

### 5.3 撤销知识库分享
10. **撤销共享**：知识库所有者可将已共享的知识库撤销共享，恢复为部门可见或私密。

### 5.4 删除知识库
11. **删除私密知识库**：普通员工可删除自己的私密知识库。
12. **批量删除**：管理员可批量删除任意知识库；部门管理员可删除非公开知识库。
13. **权限限制**：删除操作有严格的权限校验。

---

## 六、会话管理

1. **创建会话**：选择智能体（或直连模式）+ 多个知识库，创建新会话。
2. **私密会话**：创建时可勾选"私密会话"，仅创建者可见。
3. **会话列表**：左侧面板显示会话历史，按更新时间倒序排列。
4. **切换会话**：点击会话列表可切换，加载历史消息。
5. **会话隐私切换**：会话所有者可随时在"公开"和"私密"之间切换。
6. **删除会话**：会话所有者可删除自己的会话（软删除）。
7. **会话绑定知识库**：会话创建时绑定的知识库 ID 持久化存储。
8. **会话信息展示**：聊天区头部显示会话 ID、隐私状态、智能体名称、知识库数量。

---

## 七、问答/聊天功能

1. **发送问题**：底部输入框输入问题，点击"发送"或按 Enter 键提交。
2. **智能体模式**：通过 RAGFlow Agent API 进行问答，支持上下文连贯对话。
3. **直连 GPT 模式**：直接调用 OpenAI 兼容的 Chat Completions API，支持多轮对话上下文（最近 12 条历史）。
4. **多 API 格式兼容**：自动尝试多种 API 端点格式（标准 OpenAI、Azure OpenAI 等），自动适配认证方式。
5. **消息气泡**：用户消息（右侧蓝色）和助手回答（左侧灰色）以聊天气泡形式展示。
6. **长消息折叠/展开**：超过 500 字符或 8 行的长消息自动折叠，可点击"展开/收起"。
7. **错误处理**：请求失败时在助手气泡中显示错误信息。

---

## 八、引用/参考来源展示

1. **引用来源提取**：后端递归解析 RAGFlow 响应（最深 8 层嵌套），提取 chunks 引用数据。
2. **引用计数显示**：助手回答下方显示"引用来源（N 条）"的可折叠区域。
3. **引用详情**：展开后显示每条引用的文档名称和引用内容片段。
4. **格式兼容**：同时支持 dict 格式（agent session）和 list 格式（chat session）。
5. **引用持久化**：引用数据随消息存储到数据库，历史查看时可再次展示。

---

## 九、管理员专属功能

1. **资源策略管理**：查看/修改所有资源的归属部门、可见性、白/黑名单等策略。
2. **批量初始化**：一键根据智能体名称推断部门归属；一键将所有知识库设为公共 A 类。
3. **审计日志查询**：可按部门、用户、操作类型、状态筛选，支持分页。
4. **分享申请管理**：查看、批准、驳回、删除知识库分享申请。
5. **前端权限自适应**：管理员登录后侧边栏显示"分享申请"入口，普通用户隐藏。

---

## 十、审计与日志

1. **全操作审计**：所有关键操作均写入审计日志。
2. **审计字段**：记录 request_id、时间、用户 ID、部门 ID、操作类型、资源类型/ID、状态、错误信息。
3. **审计查询 API**：`GET /portal/v1/audit/logs`，支持多条件筛选和分页。

---

## 十一、前端 UI/UX 特性

1. **登录页面**：居中卡片式登录界面，显示系统名称和演示账号提示。
2. **工作台布局**：顶部导航栏 + 左侧可折叠侧边栏 + 右侧聊天主区域。
3. **侧边栏折叠/展开**：可通过按钮收起/展开，带滑动动画。
4. **移动端适配**：移动端显示汉堡菜单，点击打开侧边栏遮罩层。
5. **可折叠面板**：侧边栏内"会话历史"、"工作台设置"、"知识库管理"三个面板均可独立折叠/展开。
6. **弹窗系统**：6 个功能弹窗（创建会话、上传知识库、分享知识库、撤销分享、删除知识库、分享申请管理）。
7. **资源标签系统**：智能体和知识库旁显示彩色标签，标识归属部门和可见性。
8. **全选/全不选**：知识库选择面板提供快捷按钮。
9. **品牌标识**：顶部导航栏显示西门子医疗（Siemens Healthineers）Logo。
10. **Token 自动刷新**：页面加载时自动检查 Token 有效性，过期退回登录页。

---

## 十二、数据存储与架构

1. **SQLite 本地数据库**：所有数据存储于本地 SQLite 文件。
2. **8 张核心数据表**：`users`、`departments`、`user_departments`、`resource_policies`、`portal_sessions`、`portal_messages`、`kb_share_requests`、`audit_logs`。
3. **数据库迁移**：支持向后兼容的自动迁移。
4. **数据清理**：初始化时自动清理不在白名单中的多余用户。
5. **RAGFlow 后端集成**：通过 RAGFlow SDK 和 REST API 与 RAGFlow 服务交互。

## 十四、其他特性

1. **CORS 支持**：所有接口响应头包含跨域访问配置。
2. **请求 ID 追踪**：支持通过 `X-Request-ID` 头传递请求追踪 ID。
3. **标准化响应格式**：所有 API 返回 `{"ok": true/false, "data": ..., "error_code": ..., "message": ...}` 格式。
4. **环境变量配置**：所有关键参数通过环境变量配置（`RAGFLOW_API_KEY`、`PORTAL_AUTH_SECRET`、`PORTAL_DB_PATH` 等）。
